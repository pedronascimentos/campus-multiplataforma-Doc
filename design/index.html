    Documento de Arquitetura¶

    A arquitetura do sistema Campus Multiplataforma foi projetada para suportar um aplicativo de jornalismo multimídia desenvolvido para o Laboratório de Jornalismo do Departamento de Comunicação da Universidade de Brasília. Seu objetivo central é unificar e centralizar conteúdos de várias plataformas (LinkedIn, Instagram, Twitter, TikTok, YouTube) em um único aplicativo, além de permitir a publicação de notícias próprias no formato NewsML.

    O sistema utiliza React Native com Expo para o desenvolvimento do front-end mobile e Firebase como principal provedor de serviços em nuvem, como autenticação, armazenamento e banco de dados. Outras APIs externas são integradas para coletar e exibir conteúdo multimídia, como YouTube Data API v3, Instagram Graph API, TikTok API, Twitter API e Imgur API.

    A arquitetura foi projetada com os princípios de simplicidade, modularidade e escalabilidade, aproveitando o paradigma serverless com Firebase para reduzir a complexidade de gerenciamento de infraestrutura.

    Diagrama Geral da Arquitetura¶

    O diagrama a seguir representa a arquitetura proposta para o sistema Campus Multiplataforma:

    Componentes Principais:¶

    1. Front-End Mobile:
    2. Desenvolvido com React Native Expo.
    3. Funcionalidade de interface do usuário.

    4. Comunicação direta com Firebase e APIs externas.

    5. Backend Serverless (Firebase):

    6. Autenticação via Firebase Authentication.
    7. Armazenamento de notícias e conteúdos no formato NewsML no Firestore Database.

    8. Gerenciamento de arquivos (imagens) no Firebase Storage.

    9. Integração de APIs Externas:

    10. YouTube Data API v3: Hospedagem e recuperação de vídeos de notícias.
    11. Instagram Graph API: Incorporação de postagens (imagens e vídeos curtos).
    12. TikTok API: Incorporação de vídeos curtos.
    13. Twitter API: Incorporação de tweets (imagens, vídeos e textos).

    14. Imgur API: Hospedagem e recuperação de imagens externas.

    15. Formato NewsML:

    16. Todas as notícias, sejam originadas no app ou coletadas via APIs externas, serão convertidas e armazenadas no formato NewsML.

    Descrição Detalhada dos Componentes¶

    Front-End Mobile¶

    • Tecnologia: React Native com Expo e TypeScript.
    • Função Principal: Interface do usuário para consumir conteúdos e interagir com o sistema.
    • Detalhes Técnicos:
    • Comunicação com Firebase para acessar e armazenar dados.
    • Incorporação de conteúdos de APIs externas (YouTube, Instagram, TikTok, Twitter) com integração direta via requisições HTTP.
    • Apresentação dos dados formatados em NewsML ao usuário de forma amigável.
    • Integração com o Firebase Authentication para gerenciar usuários e permissões.

    Backend Serverless (Firebase)¶

    • Tecnologia: Firebase (Firestore, Storage, Authentication).
    • Função Principal: Fornecer uma infraestrutura escalável e sem servidor para gerenciar dados e usuários.
    • Detalhes Técnicos:
    • Firebase Authentication: Autenticação de usuários para acesso ao app.
    • Cloud Firestore: Armazenamento dos dados das notícias em formato NewsML, facilitando consultas e atualizações.
    • Firebase Storage: Armazenamento de imagens internas do sistema.

    Integração com APIs Externas¶

    O sistema incorpora conteúdos externos usando as seguintes APIs:

    • YouTube Data API v3:
    • Função: Recuperar vídeos hospedados no YouTube e integrá-los ao aplicativo.

    • Exemplo: Vídeos de notícias produzidos pela equipe do laboratório.

    • Instagram Graph API:

    • Função: Incorporar postagens de imagens e vídeos curtos diretamente no app.

    • Exemplo: Fotos ou vídeos de notícias compartilhados no Instagram.

    • TikTok API:

    • Função: Incorporação de vídeos curtos diretamente no app.

    • Exemplo: Notícias ou conteúdos de entretenimento produzidos no TikTok.

    • Twitter API:

    • Função: Recuperar tweets, incluindo textos, imagens e vídeos.

    • Exemplo: Notícias relevantes compartilhadas no Twitter com formato multimídia.

    • Imgur API:

    • Função: Hospedagem de imagens de forma eficiente e externa.
    • Exemplo: Imagens utilizadas para compor notícias publicadas pelo laboratório.

    Cada API será acessada via requisições HTTP, e os dados serão processados e armazenados no formato NewsML antes de serem apresentados no aplicativo.

    Aplicação do Formato NewsML¶

    O que é NewsML?¶

    O NewsML (News Markup Language) é um padrão baseado em XML desenvolvido para representar notícias em formatos estruturados e multimídia. Ele foi projetado para possibilitar o intercâmbio padronizado de informações jornalísticas entre sistemas e plataformas.

    Características Principais:
    - Suporte a textos, imagens, vídeos e metadados.
    - Compatibilidade com outros formatos como IPTC e XML.
    - Estrutura organizada e escalável para conteúdo jornalístico.

    Para entender a fundo o padrão NewsML, utilize o artigo “Produção de Notícias Multimídia para Diferentes Plataformas Utilizando o Padrão NewsML” (disponível no ./images/Producao_de_Noticias_Multimidia_para_Dif.pdf).

    Referência ABNT:
    GOMES, Rafael et al. Produção de notícias multimídia para diferentes plataformas utilizando o padrão NewsML. In: Anais do Congresso Brasileiro de Jornalismo Digital. São Paulo: ABCD Editora, 2023. Disponível em: ./images/Producao_de_Noticias_Multimidia_para_Dif.pdf.

    Uso do NewsML no Projeto¶

    Todas as notícias, sejam elas produzidas internamente ou coletadas de plataformas externas (YouTube, Instagram, Twitter, TikTok), serão formatadas no padrão NewsML. O formato permitirá:
    1. Estruturação dos dados (texto, imagem, vídeo e metadados).
    2. Compatibilidade com outros sistemas jornalísticos.
    3. Reuso dos dados em múltiplas plataformas.

    Exemplo Simplificado de NewsML no Projeto: 

    <newsItem>
  <newsHeader>
    <newsIdentifier>12345</newsIdentifier>
    <date>2024-06-16</date>
    <provider>Campus Multiplataforma</provider>
  </newsHeader>
  <contentSet>
    <text>Notícia relevante do Campus Multiplataforma...</text>
    <image src="https://imgur.com/12345" />
    <video src="https://youtube.com/video123" />
  </contentSet>
</newsItem>

    Estratégias de Monitoramento e Manutenção¶

    Seguindo a Manutenção Centrada em Confiabilidade, o sistema implementará as seguintes estratégias:

    1. Monitoramento Contínuo:
    2. Ferramentas como Firebase Performance Monitoring e Crashlytics serão usadas para rastrear problemas.

    3. Logs automatizados para identificar modos de falha em APIs externas.

    4. Redundância de Dados:

    5. Uso de cache local para conteúdos essenciais.

    6. Armazenamento de imagens críticas tanto no Imgur quanto no Firebase Storage.

    7. Facilidade de Atualizações:

    8. Atualizações OTA para o front-end.

    9. Deploy simplificado do backend no Firebase.

    10. Prevenção de Falhas Críticas:

    11. Implementação de timeouts e políticas de retentativa (retry policies) nas chamadas HTTP para APIs externas.
    12. Monitoramento de limites de taxa para evitar bloqueios (rate limiting).
