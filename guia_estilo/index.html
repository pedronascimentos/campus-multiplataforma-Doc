    Guia de Estilo Visual¶

    Este guia descreve os padrões visuais do aplicativo, alinhados à identidade do Campus Multiplataforma, com ajustes para melhorar a experiência do usuário (UX/UI). Ele visa orientar o design de interfaces consistentes e acessíveis.

    1. Introdução¶

    Este documento adapta o Manual de Identidade Visual Campus Multiplataforma 2023.1 para um contexto digital interativo, respeitando a identidade visual original, mas com ajustes para garantir legibilidade, contraste e usabilidade em telas. Ele será utilizado como referência para desenvolvedores e designers.

    2. Paleta de Cores¶

    Cores Principais¶

    As cores principais são utilizadas em elementos de destaque, como barras de navegação, botões e links importantes.

    Cor Código HEX
    Vermelho Vivo #E74C3C
    Laranja #F39C12
    Cinza Escuro #2C3E50

    Cores Secundárias¶

    Utilizadas para estados de hover, backgrounds e elementos decorativos.

    Cor Código HEX
    Bege Claro #F9F8EE
    Rosa Suave #F4BC99
    Marrom Claro #5A2C0A

    Rejeições¶

    Evitar o uso das cores muito saturadas do manual original, como #AB232E e #E66147, pois comprometem a experiência visual.

    3. Tipografia¶

    Fontes Primárias¶

    • Open Sans: Para textos corridos e parágrafos.
    • Roboto Slab: Para títulos e destaques, variando entre pesos médios e bold.

    Fontes Secundárias¶

    • Mali: Para seções mais descontraídas e elementos ilustrativos.

    Diretrizes de Uso: - Usar tamanhos relativos (rem) para garantir acessibilidade. - Evitar mais de duas fontes em uma mesma tela.

    4. Elementos Visuais¶

    Ícones e Ilustrações¶

    • Estilo: Ícones flat e minimalistas com inspiração autoral.
    • Dimensões: 24px para ícones padrão; 48px para ícones de destaque.
    • Atenção: Utilizar licenças livres ou autorais para qualquer material externo.

    Texturas e Formas¶

    • Aplicar padrões sutis (tonalidades baixas) para evitar distrações.
    • Evitar texturas pesadas que comprometam a leitura.

    6. Diretrizes de UX¶

    Contraste e Acessibilidade¶

    • Garantir contraste mínimo de 4.5:1 para textos pequenos e 3:1 para textos grandes.
    • Testar interfaces com ferramentas como Lighthouse ou Axe.

    Feedback ao Usuário¶

    • Usar notificações contextuais para ações como salvar ou enviar dados.
    • Indicar visualmente estados de carregamento e erros.

    Responsividade¶

    • Garantir compatibilidade para telas a partir de 320px de largura.
    • Utilizar grids flexíveis (CSS Grid ou Flexbox).

    Nota: Este guia deve ser revisado periodicamente para incorporar avanços em padrões de UX/UI e feedback dos usuários.
