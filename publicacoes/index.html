    Publicações¶

    Introdução¶

    Esta documentação detalha como configurar e integrar o YouTube Data API v3, Firebase e o formato NewsML no seu app, permitindo o upload de vídeos, coleta de analytics e armazenamento estruturado de dados jornalísticos. Utilizamos uma Conta de Serviço para autenticação segura e simplificada.

    Configuração Inicial¶

    1. YouTube Data API no Google Cloud¶

    1. Acesse o Google Cloud Console.
    2. Crie um novo projeto e ative a YouTube Data API v3.
    3. Configure uma Conta de Serviço:
    4. Vá para APIs e Serviços > Credenciais.
    5. Clique em Criar credenciais e selecione Conta de Serviço.
    6. Conceda as permissões necessárias para acessar a YouTube Data API.
    7. Baixe o arquivo JSON da conta de serviço para uso no código.

    Mais informações: Documentação de OAuth 2.0 - Conta de Serviço.

    2. Configuração do Firebase¶

    1. Configure o Firestore para armazenar os metadados e analytics dos vídeos.
    2. Instale as dependências Firebase no React Native: bash npm install @react-native-firebase/app @react-native-firebase/firestore @react-native-firebase/storage
    3. Siga a documentação oficial do Firebase para configurar o projeto.

    Funcionalidades e Fluxo¶

    Upload de Vídeos¶

    • Endpoint: videos.insert
    • Consumo: 1.600 unidades por upload.

    Passos: 1. O jornalista seleciona um vídeo no app. 2. O app autentica com a Conta de Serviço e envia o vídeo para o YouTube. 3. O ID do vídeo e metadados são armazenados no Firebase Firestore.

    Exemplo de Código:

    import axios from 'axios';

async function uploadVideo(videoFile, accessToken) {
  const formData = new FormData();
  formData.append('file', {
    uri: videoFile.uri,
    name: 'video.mp4',
    type: 'video/mp4',
  });

  const response = await axios.post(
    'https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status',
    {
      snippet: {
        title: 'Título da Matéria',
        description: 'Descrição da matéria.',
        tags: ['notícia', 'jornalismo'],
        categoryId: '22',
      },
      status: {
        privacyStatus: 'unlisted',
      },
    },
    {
      headers: {
        Authorization: `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
    }
  );

  return response.data.id; // Retorna o ID do vídeo
}

    Upload de Imagens no Imgur¶

    • Endpoint: https://api.imgur.com/3/upload
    • Autenticação: Token da conta Imgur (Bearer Token).

    Exemplo de Código:

    import axios from 'axios';

async function uploadImageToImgur(imageUri) {
  const formData = new FormData();
  formData.append('image', {
    uri: imageUri,
    name: 'image.jpg',
    type: 'image/jpeg',
  });

  const response = await axios.post(
    'https://api.imgur.com/3/upload',
    formData,
    {
      headers: {
        Authorization: 'Bearer SEU_IMGUR_TOKEN',
      },
    }
  );

  return response.data.data.link; // Retorna o link da imagem
}

    Coleta de Analytics¶

    • Endpoint: videos.list
    • Consumo: 1 unidade por vídeo.

    Passos: 1. Agende uma função para consultar métricas dos vídeos diariamente. 2. Atualize os metadados no Firestore.

    Exemplo de Código:

    import axios from 'axios';

async function fetchVideoAnalytics(videoIds, accessToken) {
  const response = await axios.get('https://www.googleapis.com/youtube/v3/videos', {
    params: {
      id: videoIds.join(','),
      part: 'statistics',
    },
    headers: {
      Authorization: `Bearer ${accessToken}`,
    },
  });

  return response.data.items.map((item) => ({
    id: item.id,
    views: item.statistics.viewCount,
    likes: item.statistics.likeCount,
    comments: item.statistics.commentCount,
  }));
}

    Publicação de Notícias Multimídia com NewsML¶

    O NewsML permite criar notícias estruturadas com suporte para texto, imagens e vídeos. Vamos usar o Firebase Storage para armazenar arquivos XML do NewsML e os recursos multimídia.

    Formato de Exemplo NewsML:

    <NewsItem>
  <Identification>
    <NewsIdentifier>unique-news-id</NewsIdentifier>
  </Identification>
  <NewsComponent>
    <Headline>Título da Notícia</Headline>
    <Text>
      <Paragraph>Este é o conteúdo textual da notícia.</Paragraph>
    </Text>
    <ContentItem MediaType="image" Source="https://i.imgur.com/imagem.jpg"/>
    <ContentItem MediaType="video" Source="https://www.youtube.com/watch?v=video-id"/>
  </NewsComponent>
</NewsItem>

    Upload de Arquivo NewsML:

    import storage from '@react-native-firebase/storage';

async function uploadNewsML(fileUri, newsId) {
  const ref = storage().ref(`/news-xml/${newsId}.xml`);
  await ref.putFile(fileUri);
  return ref.getDownloadURL(); // Retorna a URL do arquivo
}

    Salvar Metadados no Firestore:

    import firestore from '@react-native-firebase/firestore';

async function saveNewsMetadata(newsId, metadata) {
  await firestore().collection('news').doc(newsId).set(metadata);
}

const metadata = {
  title: 'Título da Notícia',
  author: 'Jornalista X',
  createdAt: new Date(),
  contentUrl: 'URL do arquivo XML',
};

saveNewsMetadata('unique-news-id', metadata);

    Leitura de Notícias no App¶

    Fluxo: 1. O app recupera os metadados e links de mídia do Firestore. 2. Renderiza o conteúdo textual e carrega as mídias vinculadas.

    Exemplo de Renderização:

    import React from 'react';
import { Text, Image, View } from 'react-native';
import YoutubePlayer from 'react-native-youtube-iframe';

function NewsArticle({ newsData }) {
  return (
    <View>
      <Text>{newsData.title}</Text>
      <Text>{newsData.content}</Text>
      <Image source={{ uri: newsData.image }} style={{ width: 300, height: 200 }} />
      <YoutubePlayer height={200} play={false} videoId={newsData.videoId} />
    </View>
  );
}

    Coleta de Analytics 1/unidade 400 (para 400 vídeos) Estatísticas de 400 vídeos gerenciados

    Prós e Contras¶

    Prós¶

    • Gratuito: O YouTube fornece armazenamento e streaming otimizados sem custo adicional.
    • Simplicidade: APIs bem documentadas para upload e gerenciamento.
    • Escalabilidade: Suporte a grandes volumes de vídeos com coleta eficiente de analytics.

    Contras¶

    • Limitação de Cota: Restrição de 10.000 unidades diárias pode ser um gargalo em alta demanda.
    • Responsabilidade Centralizada: Todos os vídeos estão vinculados a uma única conta.
    • Privacidade Limitada: Dependência das configurações de privacidade do YouTube.

    Para mais informações, consulte a documentação oficial da YouTube Data API.
