    Imgur API¶

    Introdução¶

    A configuração e integração da Imgur API permite o upload de imagens e gerenciamento de galerias de forma simples e eficiente. Utilizando autenticação via OAuth 2.0, a API possibilita que os jornalistas façam o upload de imagens diretamente para o aplicativo.

    Essa documentação aborda a integração do Imgur API para upload de imagens, compressão/descompressão nativa de imagens no app e como integrar essas imagens em um formato NewsML, analisando também o impacto na performance.

    A seguir, a configuração do ambiente, as funcionalidades de upload e leitura de dados, além de estimativas do uso da API para análise de viabilidade. Para mais informações, consulte a documentação oficial da Imgur API.

    Configuração Inicial¶

    Criação de Conta Imgur¶

    1. Acesse o site do Imgur e crie uma conta.
    2. Registre seu aplicativo no Portal de Desenvolvedores da Imgur para obter o Client ID e Client Secret.

    Obtenção do Token de Acesso¶

    Para autenticação via OAuth 2.0, siga o processo para obter um Access Token com permissões adequadas. Mais detalhes no guia de autenticação da Imgur.

    Funcionalidades da Imgur API¶

    Upload de Imagens¶

    • Endpoint: POST https://api.imgur.com/3/upload
    • Autenticação: Adicione o Access Token no cabeçalho da requisição.

    Exemplo de Código para Upload Público:¶

    import axios from 'axios';

async function uploadImage(imageUri) {
  const formData = new FormData();
  formData.append('image', {
    uri: imageUri,
    name: 'image.jpg',
    type: 'image/jpeg',
  });

  const response = await axios.post('https://api.imgur.com/3/upload', formData, {
    headers: {
      Authorization: 'Bearer SEU_ACCESS_TOKEN',
    },
  });

  return response.data.data.link; // Retorna o link público
}

    Exemplo de Código para Upload Privado:¶

    async function uploadPrivateImage(imageUri) {
  const formData = new FormData();
  formData.append('image', {
    uri: imageUri,
    name: 'image.jpg',
    type: 'image/jpeg',
  });

  const response = await axios.post('https://api.imgur.com/3/upload', formData, {
    headers: {
      Authorization: 'Bearer SEU_ACCESS_TOKEN',
    },
    params: {
      privacy: 'hidden', // Torna a imagem privada
    },
  });

  return response.data.data.link; // Retorna o link privado
}

    Coleta de Informações sobre Imagens¶

    • Endpoint: GET https://api.imgur.com/3/account/{username}/images
    • Autenticação: Adicione o Access Token.

    Exemplo de Código:¶

    async function fetchUserImages(username) {
  const response = await axios.get(`https://api.imgur.com/3/account/${username}/images`, {
    headers: {
      Authorization: 'Bearer SEU_ACCESS_TOKEN',
    },
  });

  return response.data.data; // Retorna as imagens do usuário
}

    Compressão e Descompressão de Imagens Nativa¶

    Compressão de Imagens¶

    Utilize a biblioteca react-native-image-crop-picker para comprimir as imagens diretamente no dispositivo, reduzindo seu tamanho sem perder qualidade excessiva.

    import ImagePicker from 'react-native-image-crop-picker';

async function compressImage(imageUri) {
  const compressedImage = await ImagePicker.openCropper({
    path: imageUri,
    width: 800,
    height: 800,
    compressImageQuality: 0.7, // Ajuste da compressão
  });

  return compressedImage.path;
}

    Descompressão de Imagens¶

    Para restaurar a imagem ao seu tamanho original após compressão, use react-native-image-resizer.

    import ImageResizer from 'react-native-image-resizer';

async function decompressImage(compressedUri) {
  const decompressedImage = await ImageResizer.createResizedImage(
    compressedUri,
    1600, // Largura
    1600, // Altura
    'JPEG',
    100   // Qualidade
  );

  return decompressedImage.uri;
}

    Impacto na Performance¶

    • Compressão: A compressão de imagens no dispositivo pode aumentar o uso de CPU, especialmente em dispositivos com hardware limitado. Use técnicas assíncronas para não bloquear a interface do usuário.
    • Descompressão: Similarmente, a descompressão de imagens pode exigir recursos significativos, mas a diferença de performance será menos perceptível em imagens menores ou dispositivos mais potentes.

    Recomenda-se medir o tempo de processamento para garantir que a compressão/descompressão não afete negativamente a experiência do usuário.

    Integração com NewsML¶

    Formato NewsML¶

    O NewsML é um formato padrão para armazenamento e troca de notícias, utilizado por diversas organizações de mídia. A seguir, mostramos como integrar as imagens enviadas para o Imgur ou Firebase no formato NewsML.

    Exemplo de NewsML com Imgur:¶

    <NewsItem>
  <Identification>
    <NewsIdentifier>unique-news-id</NewsIdentifier>
  </Identification>
  <NewsComponent>
    <Headline>Título da Notícia</Headline>
    <Text>
      <Paragraph>Este é o conteúdo da notícia.</Paragraph>
    </Text>
    <ContentItem MediaType="image" Source="https://i.imgur.com/imagem.jpg"/>
  </NewsComponent>
</NewsItem>

    Exemplo de NewsML com Firebase Storage:¶

    <NewsItem>
  <Identification>
    <NewsIdentifier>unique-news-id</NewsIdentifier>
  </Identification>
  <NewsComponent>
    <Headline>Título da Notícia</Headline>
    <Text>
      <Paragraph>Este é o conteúdo da notícia.</Paragraph>
    </Text>
    <ContentItem MediaType="image" Source="https://firebasestorage.googleapis.com/v0/b/seu-app.appspot.com/o/imagem.jpg?alt=media"/>
  </NewsComponent>
</NewsItem>

    Upload de Arquivo NewsML¶

    Você pode usar Firebase Storage ou qualquer outro serviço para armazenar e enviar arquivos NewsML.

    Estimativas de Uso¶

    • Cotas Gratuitas Imgur:
    • 50 uploads diários.

    • Limite de Tamanho: 10 MB por imagem.

    • Cotas Gratuitas Firebase:

    • 5 GB de armazenamento e 1 GB de download mensal gratuito.
    • Após ultrapassar esses limites, você será cobrado de acordo com os preços do Firebase.

    Prós e Contras¶

    Prós¶

    • Imgur: Simples de usar, permite tanto uploads públicos quanto privados, e oferece uma boa quantidade de uploads diários gratuitamente.
      • Gratuito: O Imgur fornece armazenamento de imagens sem custo adicional.
      • Simplicidade: API fácil de integrar e bem documentada.
      • Escalabilidade: Suporte a grandes volumes de imagens.
    • Firebase: Excelente para armazenamento escalável e fácil integração com outras funcionalidades do Firebase.

    Contras¶

    • Imgur: Limitação de 50 uploads diários e tamanho de 10 MB por imagem.
      • Limitação de Cota: A cota gratuita pode ser limitada.
      • Privacidade Limitada: Dependência das configurações de privacidade do Imgur.
    • Firebase: Pode ser caro para grandes volumes de armazenamento ou downloads frequentes.

    Para mais informações, consulte a documentação oficial da Imgur API.
